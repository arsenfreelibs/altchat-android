# Android App — Architecture

## Обзор

Приложение построено на основе **deltachat-core-rust** — Rust-библиотеки, реализующей всю протокольную логику (IMAP/SMTP, e2e-шифрование, управление аккаунтами). Android-слой отвечает за UI, интеграцию с системой и взаимодействие с core через два механизма: **JNI-обёртки** (legacy) и **JSON-RPC поверх FFI** (актуальный).

## Описание слоёв

### 1. Native / Core Layer

| Что | Где |
|---|---|
| Rust-библиотека DeltaChat Core | `jni/deltachat-core-rust/` |
| C JNI-glue (мост между Java и Rust через `deltachat-ffi`) | `jni/dc_wrapper.c` |
| Скомпилированные `.so` под 4 ABI | `libs/{arm64-v8a,armeabi-v7a,x86,x86_64}/` |

Реализует: IMAP/SMTP IO, e2e-шифрование (Autocrypt), SecureJoin/QR-invite, работу с аккаунтами, базой сообщений, вложениями, геолокацией, webxdc, звонками.

**Принцип**: этот слой не знает ничего об Android. Вся логика — только в Rust.

---

### 2. JNI Bridge Layer — `com.b44t.messenger`

Тонкие Java-обёртки над нативными структурами. Держат `long *CPtr` — указатели на Rust-объекты.

| Класс | Назначение |
|---|---|
| `DcAccounts` | Менеджер всех аккаунтов; `startIo()`, `stopIo()`, `backgroundFetch()` |
| `DcContext` | Один аккаунт: настройки, чаты, контакты, сообщения (вся legacy API) |
| `DcChat` | Данные одного чата |
| `DcMsg` | Данные одного сообщения |
| `DcContact` | Данные контакта |
| `DcEvent` | Событие от core |
| `DcEventEmitter` | Получение событий из native (legacy, заменён `DcEventChannel`) |
| `DcEventChannel` | Канал событий для `DcAccounts` |
| `DcJsonrpcInstance` | JSON-RPC endpoint поверх native (`request()` / `getNextResponse()`) |
| `DcChatlist`, `DcLot`, `DcProvider` | Вспомогательные структуры |
| `FFITransport` | Реализует `Rpc.Transport` через `DcJsonrpcInstance` |

**Принцип**: только маппинг native → Java. Никакой Android-специфичной логики.

---

### 3. RPC Layer — `chat.delta.rpc`

Автогенерированный (не редактировать вручную) JSON-RPC клиент.

| Класс | Назначение |
|---|---|
| `Rpc` | Методы API (autogenerated): `addAccount`, `sendMsg`, `getChatMsgs`, `getFullChat`, … |
| `BaseTransport` | Базовая реализация транспорта: очередь запросов, `Future`-based ответы |
| `FFITransport` (в `com.b44t.messenger`) | Конкретный транспорт поверх `DcJsonrpcInstance` |
| `RpcException` | Исключение RPC |
| `chat.delta.rpc.types.*` | DTO: `Message`, `FullChat`, `Contact`, `Account`, `Event`, `Qr`, … |

**Принцип**: `Rpc` — предпочтительный способ вызова core API для новых фич. `DcContext` используется там, где RPC-метод ещё не добавлен.

---

### 4. Application Layer

**`ApplicationContext`** (`org.thoughtcrime.securesms.ApplicationContext`)
Точка входа приложения (`MultiDexApplication`). Singleton. Хранит и инициализирует:
- `DcAccounts` (static, thread-safe initialization lock)
- `Rpc` (через `FFITransport(DcAccounts.getJsonrpcInstance())`)
- `DcContext` (текущий аккаунт, может меняться через `setDcContext`)
- `DcEventCenter`
- `NotificationCenter`
- `DcLocationManager`
- `JobManager`

**`connect/` — пакет управления подключением и аккаунтами**

| Класс | Назначение | Что может вызывать |
|---|---|---|
| `DcHelper` | Утилитный фасад: статические геттеры `getContext()`, `getRpc()`, `getAccounts()`, конфиги | `ApplicationContext`, `DcContext`, `Rpc` |
| `AccountManager` | Переключение / миграция аккаунтов | `DcAccounts`, `DcHelper`, `ApplicationContext` |
| `DcEventCenter` | Observer для событий core (`addObserver`, `removeObserver`) | `DcEvent`, `ApplicationContext` |
| `FetchWorker` | `WorkManager.Worker` — периодическая фоновая синхронизация | `DcAccounts.startIo()` |
| `ForegroundDetector` | Отслеживает, в foreground ли приложение | только читается |
| `KeepAliveService` | Foreground service — держит IO живым | `DcAccounts` |
| `NetworkStateReceiver` | `BroadcastReceiver` — реагирует на смену сети | `DcAccounts.maybeNetwork()` |

**`notifications/`**

| Класс | Назначение |
|---|---|
| `NotificationCenter` | Показывает системные уведомления (входящие сообщения, звонки) |
| `InChatSounds` | Звуки в чате |
| `MarkReadReceiver` | BroadcastReceiver для пометки сообщений прочитанными |
| `RemoteReplyReceiver` | BroadcastReceiver для ответа из уведомления |
| `DeclineCallReceiver` | BroadcastReceiver для отклонения звонка |

**`service/`**

| Класс | Назначение |
|---|---|
| `FetchForegroundService` | Foreground service для фоновой загрузки сообщений |
| `GenericForegroundService` | Базовый foreground service |
| `AudioPlaybackService` | Воспроизведение аудио (media3) |
| `BootReceiver` | Запускает IO после перезагрузки устройства |

**`jobmanager/`** — очередь задач с WakeLock-поддержкой (legacy, для тяжёлых background-операций).

**`crypto/`** — `DatabaseSecret`, `DatabaseSecretProvider`, `KeyStoreHelper`: хранение ключа шифрования БД вложений в Android Keystore.

**`database/`** — только `AttachmentDatabase` (enum констант статусов) + `Address`/`ThreadRecord`. Основная БД — в Rust core.

**`recipients/`** — `Recipient`: Android-side модель получателя: объединяет `DcContact` + `DcChat`, вычисляет аватар.

**`geolocation/`** — `DcLocationManager`, `DcLocation`, `LocationBackgroundService`: трекинг GPS-локации и отправка в чат.

### 5. UI Layer

**Основные Activity**

| Класс | Назначение |
|---|---|
| `ConversationListActivity` | Список чатов (главный экран) |
| `ConversationActivity` | Экран чата (ввод + список сообщений) |
| `ConversationFragment` | RecyclerView сообщений |
| `GroupCreateActivity` | Создание группы |
| `ProfileActivity` | Профиль контакта |
| `ContactSelectionActivity` | Выбор контактов |
| `WebxdcActivity` | WebView для webxdc-приложений |
| `CallActivity` | WebRTC-звонок (WebView + JS bridge) |
| `QrActivity` / `QrShowActivity` | Сканирование / отображение QR |
| `BackupTransferActivity` | Бэкап/восстановление аккаунта |
| `InstantOnboardingActivity` | Онбординг нового аккаунта |
| `WelcomeActivity` | Приветственный экран |
| `ApplicationPreferencesActivity` | Настройки |
| `LogViewActivity` | Просмотр логов |

**Фрагменты** (`*Fragment`) — смонтированы внутри Activity.

**Адаптеры** (`*Adapter`) — `RecyclerView.Adapter` для списков чатов, сообщений, контактов.

**Специализированные**:
- `search/` — поиск по сообщениям
- `reactions/` — реакции
- `relay/` — пересылка сообщений
- `proxy/` — настройка прокси
- `scribbles/` / `imageeditor/` — редактор изображений
- `audio/` — аудиозаписи
- `video/` — видео

## Правила вызовов между слоями

```
UI Layer
  ├── МОЖЕТ вызывать: Application Layer (DcHelper, AccountManager, DcEventCenter)
  ├── МОЖЕТ вызывать: Rpc (chat.delta.rpc) — напрямую или через DcHelper.getRpc()
  ├── МОЖЕТ вызывать: JNI Bridge (DcContext, DcChat, DcMsg, DcContact) — только read
  └── НЕ ДОЛЖЕН вызывать: Native/Core Layer напрямую

Application Layer
  ├── МОЖЕТ вызывать: JNI Bridge Layer (DcAccounts, DcContext, DcEvent*)
  ├── МОЖЕТ вызывать: RPC Layer (Rpc)
  └── НЕ ДОЛЖЕН вызывать: UI Layer (нет прямых зависимостей на Activity/Fragment)

RPC Layer
  ├── МОЖЕТ вызывать: JNI Bridge Layer (только через Transport интерфейс)
  └── НЕ ДОЛЖЕН вызывать: Application Layer / UI Layer

JNI Bridge Layer
  ├── МОЖЕТ вызывать: Native/Core Layer (через JNI/FFI)
  └── НЕ ДОЛЖЕН вызывать: Application Layer / UI Layer / RPC Layer
      (исключение: FFITransport реализует Rpc.Transport — это нормально)

Native/Core Layer
  └── Ничего не знает об Android. Только Rust/C.
```

## Поток событий (Event Flow)

```
Rust Core
    │  dc_wrapper.c (JNI callbacks)
    ▼
DcEventChannel / DcEventEmitter
    │
    ▼
ApplicationContext.onEvent()
    │
    ├──► DcEventCenter.emit(event)
    │        └──► все зарегистрированные DcEventDelegate.handleEvent()
    │             (Activity / Fragment / NotificationCenter)
    │
    └──► NotificationCenter (для входящих сообщений + звонков)
```

| `gplay` | `org.thoughtcrime.securesms` / Push через FCM | Firebase Cloud Messaging, Google Play Services |

## Ключевые зависимости (build.gradle)

- `deltachat-core-rust` — Rust core (компилируется через NDK)
- `jackson-databind` — JSON сериализация для RPC
- `glide` — загрузка изображений / аватаров
- `media3` — воспроизведение аудио/видео
- `androidx.work` — WorkManager для FetchWorker
- `signal-protocol` — криптографические примитивы (legacy)
- `zxing` / `journeyapps` — QR-код сканер

## Итого: где что делается

| Задача | Слой | Класс(ы) |
|---|---|---|
| Отправить сообщение | RPC | `Rpc.sendMsg()` |
| Получить список чатов | RPC | `Rpc.getChatlistItemsByEntries()` |
| Переключить аккаунт | Application | `AccountManager.switchAccount()` |
| Подписаться на событие | Application | `DcEventCenter.addObserver()` |
| Показать уведомление | Application | `NotificationCenter.updateNotification()` |
| Запустить IO (синхронизацию) | Application | `DcAccounts.startIo()` (через DcHelper) |
| Данные чата/контакта | JNI Bridge | `DcContext.getChat()`, `DcContact` |
| Конфиг аккаунта | JNI Bridge / DcHelper | `DcHelper.get(context, key)` |
| Отобразить чат | UI | `ConversationActivity` + `ConversationFragment` |
| Обработать QR | UI + RPC | `QrCodeHandler` + `Rpc.checkQr()` |
| Webxdc | UI + RPC | `WebxdcActivity` + `Rpc.getWebxdcInfo()` |
| Звонок | UI + RPC | `CallActivity` + `Rpc.*Call*()` |
